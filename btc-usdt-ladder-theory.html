1. 核心模型

小币合约空头模型关心的是空头仓位。BTC/USDT 模型关心的是资产结构：你手里有多少 BTC，多少 USDT，以及这些仓位分别承担什么任务。

价格 P + 历史高点 ATH + 回撤状态 D -> BTC 仓位 B 与 USDT 仓位 U B, U = f(P, ATH, D, R)

这里的目标不是“卖在最高点”，而是：上涨进入狂热区时降低战术风险，下跌进入折扣区时恢复或增加 BTC 数量。

2. ATH 不是顶部

ATH 是历史最高点，不是未来最高点。牛市真正强的时候，突破旧 ATH 往往意味着进入价格发现区。到 ATH 就全卖，最大的风险不是亏钱，而是被趋势甩下车。

接近旧 ATH：提醒 突破 ATH：战术减仓 高点回撤：分层买回

所以更合理的处理是：旧 ATH 附近提醒，ATH 上方分层卖出战术仓，而不是把所有 BTC 一次性换成 USDT。

3. 核心仓与战术仓

如果你的核心目标是囤 BTC，仓位最好分成两层。

核心 BTC 仓

长期不动，例如 60%-80%。它负责穿越周期，避免你在真正的大趋势里彻底踏空。

战术 BTC 仓

用于高位分层换成 USDT、深跌分层买回 BTC。它负责改善周期内的风险和现金流。

总 BTC = 核心 BTC + 战术 BTC 核心 BTC：不参与 ATH 清仓 战术 BTC：按价格阶梯卖出，按回撤阶梯买回

4. ATH 上方卖出阶梯

卖出阶梯可以用旧 ATH 作为锚点，再按百分比往上展开。每一层只卖战术仓的一部分。

卖出层： S_i = ATH * (1 + a)^i 例：ATH = 126,080, a = 15% 126,080 / 144,992 / 166,741 / 191,752

这样做的含义是：越进入价格发现区，越把部分战术 BTC 转成 USDT，但核心仓继续保留上行参与权。

5. 高点回撤买回阶梯

买回不应该只看旧 ATH，而应该看突破后形成的阶段高点。因为如果 BTC 从 126k 涨到 180k，再跌回 150k，它已经回撤不少，但仍高于旧 ATH。

买回层： B_i = Peak * (1 - d * i) 例：阶段高点 Peak = 180,000, d = 18% 147,600 / 115,200 / 82,800 / 50,400

如果之前在高位卖出 0.05 BTC 得到 USDT，之后用同一笔 USDT 在更低价买回，就可能买回超过 0.05 BTC。模型的关键是用“已卖出的战术资金”买回，而不是无限加仓。

6. 风险边界

BTC/USDT 阶梯再平衡比合约做空温和，但不是无风险。它的主要风险是踏空、买不回来、USDT 与交易所风险、税务和手续费。
