## BTC 特有的复核字段(Q2 补充)

每次复核除了通用 5 类,还要过这些:

- **货币化进程**:主权 / 央行 / 机构持有比例;ETF 流入流出趋势
- **网络数据**:hashrate / 持币地址分布 / on-chain volume / Lightning 容量
- **监管环境**:美国 / 欧盟 / 中国 / 香港 / 中东 主要司法管辖区的政策方向
- **macro 背景**:M2 / 实际利率 / 美元指数 / 黄金价格 — BTC 作为通胀对冲 / 数字黄金叙事是否在 working
- **减半周期位置**:距离上次 / 下次减半多久,历史周期模式是否还成立(或已经被 ETF 资金流打破)

## 第一次 re-underwrite 时必须回答的"为什么 BTC 仓 < MSTR 仓"(3 选 1)

1. **便利性 trap** — BTC 自托管 / 交易所 / 税务麻烦,所以买股票替代了。conviction 输给了 friction
2. **隐含 thesis 升级** — 嘴上 BTC > MSTR 但其实信飞轮跑赢 base case → 校准排序而非仓位
3. **历史路径锁定** — 几年前不同 mNAV 时分批建的仓,从来没 re-underwrite 过

写下来你判断是哪种,以及打算不打算调整。
