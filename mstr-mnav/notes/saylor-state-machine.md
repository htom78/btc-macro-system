## 1. 核心命题

Saylor 在玩一场**溢价对冲游戏**:
- 高溢价 → ATM 砸股票卖给市场,换被低估的 BTC
- 低溢价 → 转债加杠杆(零息),把杠杆补回去
- 滚动展期,把刚性还债日永远推到下一个 BTC 牛市之后

所以 MSTR 不是被动跟随 BTC 的标的,**Saylor 自己就是 MSTR 最大的边际空头,也是 BTC 最大的边际多头**。
现有 mnav-monitor 只画 mNAV 曲线,没刻画"Saylor 此刻在踩油门还是踩刹车" —— 这一层是产品叙事的缺口。

---

## 2. 为什么有溢价(三类买家结构)

理解 S1/S2 切换之前,先理解**溢价从哪来**。郑迪给出了机构资金的三层决策结构 ——
任何一类买家退场都会压溢价,任何一类回流都会撑溢价。这是 mNAV 数字背后的物理。

### 第一层 · 合规绕道买家(底部硬支撑)

不是不想买 BTC ETF,是**不能买**:

| 谁 | 卡在哪 |
|---|---|
| 国内 TRS 用户(人民币产品) | 国内券商 TRS 通道**禁止**买 BTC ETF,但 crypto-related 美股(MSTR / MARA / COIN)放行 |
| 韩国养老金 | mandate 不允许直接买 ETF |
| 美国州养老金 | manager 历史授权清单里没 BTC ETF,改 mandate 要走内部流程 |
| 欧洲保险等海外机构 | 同上 |

→ MSTR 是这群人**唯一能合规拿到的 BTC 敞口代理**。
**这一层不会因为 mNAV < 1 跑路** —— 他们没别的选择。**所以这是 MSTR 估值底部的硬锚**。

### 第二层 · 杠杆买家(放大牛市,压回熊市)

即便能买 IBIT,大资金也愿意来 MSTR,因为 Saylor 替你把杠杆加好了:

- **你自己没加杠杆** → 没 margin call,没强平
- **场外加杠杆** → margin call 风险悬头上
- **MSTR 上涨幅度 > BTC**(Saylor 自己 PPT 反复演示)→ 牛市跑赢 IBIT

**这一层在 S2(砸盘)阶段被打压,在 S1(加杠杆)阶段回流**。
是 mNAV 1.0-2.0 区间的主要力量。

### 第三层 · 期权深度买家(对冲基金,溢价的真正推手)

最少人讲、但最关键 —— 解释了**为什么 mNAV 能炒到 100%+ 溢价**:

> MSTR 一只股票的期权成交量 > 所有 BTC-related 股票(COIN / MARA / RIOT 等)期权之和

意味着:
- 想用 Black-Scholes 框架押 BTC 波动率,**市面上没有第二个标的有这种深度**
- MSTR 实质上 = **隐含的 BTC 看涨期权 + 永久流动性**
- 波动率↑ → 期权 IV↑ → 溢价合理化

**这一层是 S2 高溢价的真正定价者**。一旦波动率压下来(熊市 / 震荡市),溢价 60% → 0% 就发生在这群人撤场的几周内。

### 这三层怎么进 dashboard

不强行做出"今天哪层在退场"的判定(数据成本太高),但状态机可以借此推断:

| 当前观察 | 大概率发生了什么 |
|---|---|
| mNAV 跌破 1.0 但跌不深(0.85-1.0 反复) | 第一层在接,第二三层撤了 |
| mNAV 从 2.5 快速回 1.5 | 第三层(波动率交易者)在 unwind,见 §3 的 S2 自我打压 |
| BTC 涨但 mNAV 不动 | 第二层没回来,可能他们看到 ATM 还有大量子弹不敢追 |

**Saylor 营销战略的核心**:不断 podcast / 推特,**就是在维持第三层的注意力**(他们才是溢价的金主)。
未来如果 Saylor 减少公开露面频率,要立即调低对 mNAV 上限的预期 —— 这是叙事撤资信号。

### 为什么没人能复制 MSTR

Metaplanet / 港股微策略 / 各种"日本版微策略"全部失败的根因:**三件套凑不齐**。

| 件套 | MSTR 独占 | 模仿者状况 |
|---|---|---|
| 美股流动性 | 全球最大资金池 | 区域市场,流动性低一个量级 |
| 教主级 IP | Saylor podcast/推特/演讲全频率覆盖 | 创始人不出来路演 |
| 单一标的极端波动率 | 期权 IV 自我强化 | 标的小,期权市场撑不起 |

模仿者最多做成"莱特币级别"(BTC 市值 1-2%),复刻不了 MSTR 的 100%+ 溢价。

## 3. 三态机定义

| 状态 | 触发条件 | Saylor 在做什么 | 对 MSTR 影响 | 对 BTC 影响 |
|---|---|---|---|---|
| **S1 · 加杠杆周期**(踩油门) | mNAV < 1.5 **且** ATM 授权剩余 > 50% **且** 最近一次发行=转债/优先股 | 主用发债买 BTC,买入节奏受发行节奏(月级)拖慢 | 偏多(杠杆补回)| 边际买盘减弱,中性偏空 |
| **S2 · 砸盘去溢价**(踩刹车) | mNAV > 2.0 **且** ATM 授权使用率上升 **且** 最近一次发行=ATM 股票 | 日级别净卖出股票换 BTC | 偏空(自我打压溢价) | 偏多($亿级日均买入)|
| **S3 · 静默期** | 财报前 ~30 天 | 不能做任何融资操作 | 中性 | 失去最大边际买家,易回调 |

**状态切换信号(从 S1 → S2 的拐点)**:
- ATM 授权刚刚被董事会批复(看 8-K)
- 最大可发行股数被董事会临时扩容(2026-01 那次 3亿股 → 103亿股是经典案例)
- 季报 PPT 出现 "ATM remaining" 显著消耗

**状态切换信号(从 S2 → S1 的拐点)**:
- 季报 PPT 明确说"主要融资中心转向发债"(2026-02-04 财报)
- ATM 授权额度打到 70%+
- 加权杠杆率(BTC value / market cap)跌破自家长期目标下限(他自己 PPT 写的 20-30%,实际触发是 15% 左右)

## 4. 映射到现有 mNAV 阈值

现有 `constants.py` 的 5 档阈值是**静态分类**,把它叠加上 Saylor 状态后变成**动作建议**:

| mNAV 区段 | 单看 mNAV(现有) | 叠加 Saylor 状态后的解读 |
|---|---|---|
| ≥ 2.00 (P3) | 飞轮过度定价 | **S2 必发**:Saylor 必砸 ATM,**多头跑路信号**(优先级高于单纯 mNAV 信号) |
| 1.50-2.00 (P2) | 观察 | 看 ATM 剩余余额:>50% 危险,<20% 反而是 S1 转换信号 |
| 1.20-1.50 (P1) | 入场 | 大概率 S1,Saylor 正加杠杆,飞轮健康 |
| 1.00-1.20 | 极度便宜 | S1 末段 / 可能他在抵押 BTC 借钱(上轮 Silvergate 案例) |
| < 0.85 (P0) | 飞轮停转 | 配合上轮熊市净资产为负,看的不是 mNAV 而是 28 年到期日距离 |

## 5. 时间窗倒计时(D-Day 监控)

| 到期日 | 金额 | 强制条件 | 应监控 |
|---|---|---|---|
| 2028-03 | $20亿(转债) | 股价 < $430 → 投资人面值回购权 | BTC 价格 + MSTR 股价距 $430 的距离 |

- "距 2028-03 还有 X 天"
- "若今日清算,$20亿 缺口对应 BTC 数量 = N 枚"

## 6. 实现优先级(对现有代码的最小侵入式改动)

### P0 · constants.py 增字段

```python
# Saylor 状态机阈值(同上文 §3)
ATM_AUTHORIZATION_TOTAL_USD: float = 21_000_000_000  # 每轮 $42B 计划里股票那半
ATM_REMAINING_HIGH: float = 0.50  # >50% 余额 = S1 可持续
ATM_REMAINING_LOW: float  = 0.20  # <20% 余额 = 即将转 S2 → S1

# D-Day 倒计时
DDAY_2028_MAR_AMOUNT_USD: float = 2_000_000_000
DDAY_2028_MAR_CONVERSION_PRICE: float = 430.0
```

### P1 · monitor.py 增 classify_saylor_state()

输入:最近一次 8-K 类型、ATM 剩余、近 30 天是否财报静默
输出:`"S1" | "S2" | "S3" | "unknown"`(unknown 时 fallback 到现有 mNAV-only 逻辑)

### P2 · dashboard.py 增"Saylor 状态指示器"

页头一行:`🟢 S1 · 加杠杆中 | ATM 剩余 $62B / $210B (30%) | 距 2028-03 D-Day:752 天`

### P3 · suggest.py 增 R9/R10 规则(spec 原写 R4,现实里 R4 已被「历史百分位」占用)

```
R9:  if state == "S2" and mNAV >= 2.0:
        level = "critical"
        label = "Saylor 高溢价砸 ATM 进行中 — 多头跑路信号"

R10: if state == "S1":  # 信息性,不改 level
        cautions += "Saylor 加杠杆周期中,ATM 剩余 X% 子弹"
```

升级路径:`critical` 是新增 level,比 `warning` 更重(深酒红 #7b241c)。
仅当 Saylor 状态机能确认 S2(需要 EDGAR 数据)时才触发,unknown 状态走原有 mNAV 单点规则。

## 8. 反共识结论(产品文案可直接引用)

1. **BTC 涨 ≠ MSTR 必涨** —— S2 状态下方向反走
2. **Saylor 不会被迫卖 BTC**(3 年内没有强制还债)
3. **做空 MSTR 溢价 + 做多 BTC = 历史上死最惨**(只有 Saylor 自己知道何时砸)
4. **模仿者复制不了**(缺美股流动性 + 缺教主级 IP + 营销定位指错人群)
5. **真正的 D-Day 在 2028,不在任何单一 mNAV 阈值**

## 9. 阈值复评触发条件

**当前决策(2026-05-27)**: P3_high_premium = 2.00 暂不动。

**理由**:
1. 历史 1461 样本 p87 ≈ 2.00,本轮峰 ~2.1 仍能触发,数据未证明失效
2. `PERCENTILE_HIGH = 0.90` 已提供动态分位预警,补足绝对阈值的静态性
3. 改 P3 牵动 `suggest.py` / `monitor.py` / `saylor_state.py` 至少 6 处 classify / R9 / S2 触发逻辑,改动成本高

**重新评估的触发条件(任一满足即重启讨论)**:

- **下一个 BTC 牛市顶 mNAV 没破 1.80**
  → 说明郑迪"市场学习效应"论生效,溢价上限实质收窄
  → 届时降 P3 到 1.50-1.70,同时 R9 critical 触发条件同步下调

- **美国 BTC 战略储备正式落地**(联邦层面,不只是州层面)
  → 大型机构 mandate 集中改完,合规绕道买家底部支撑层消失
  → MSTR 溢价空间被进一步压缩,需要重新校准整套 THRESHOLDS

- **本轮 BTC 周期结束(下一个熊市底)mNAV 不再跌破 P1_extreme = 1.00**
  → 说明三类买家中至少两类已稳态化,下行波动收窄
  → 不只是 P3,P0_deep_discount = 0.85 也需要重新校准

**复评流程**: 跑一次 1461+ 全样本分位重算 → 看本轮顶/底 vs 上轮 → 决定是平移 THRESHOLDS 整套还是只动 P3
