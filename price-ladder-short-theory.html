价格阶梯仓位模型：理论基础

这不是一个“猜顶”方法，而是一套把顶部未知、开仓触发、仓位大小和最大亏损拆开的研究框架。适用场景是：小币短期暴涨后想观察潜在做空机会，但不确定价格会先冲到哪里。

仅用于策略建模和风控演示，不构成投资建议。合约、杠杆和小市值币种可能出现极端波动、滑点、资金费率变化和强平风险。

1. 核心模型

原始问题是：我认为暴涨后迟早会回落，但不知道顶部在哪里。价格阶梯模型的处理方式不是预测顶部，而是把决策改写成函数。

价格 P + 状态信号 C + 风险预算 R -> 目标空头仓位 S S = f(P, C, R)

其中，P 是当前价格，C 是确认信号，R 是可承受亏损，S 是目标空头仓位。这个框架解决四个问题：价格涨到哪一层提醒，是否需要转弱确认，每层开多少，继续上涨时在哪里认错。

2. 价格阶梯

小币合约更适合用百分比阶梯，而不是固定价格步长。因为从 30 到 40 是涨 33.3%，从 80 到 90 只是涨 12.5%，固定步长会让不同价位的风险密度失真。

百分比阶梯： L_i = P0 * (1 + r)^i 例：P0 = 30, r = 20% 36, 43.2, 51.84, 62.21, 74.65, 89.58

等差阶梯

L_i = P0 + i * Δ。适合展示概念，但在价格跨度很大时不够自然。

等比阶梯

L_i = P0 * (1 + r)^i。每一层都是同等百分比变化，更适合高波动合约。

斐波那契阶梯

L + ratio * (H - L)。用于把一个已知波段按 23.6%、38.2%、61.8% 等比例切分。

波动率阶梯

阶梯距离由 ATR、历史波动率或盘口波动决定。波动越大，层间距越宽。

3. 状态机

这个模型里，到价不等于开仓。价格层更像雷达坐标，真正开空需要状态信号确认。

未触发 已提醒 等确认 已开空 止盈 / 止损
条件开空示例： 价格触及 L_i -> 只发提醒 -> 从阶段高点回落 c% -> 或跌破短周期结构 -> 才允许开第 i 层空单

这样做的目的，是避免在单边拉升里被动连续成交。你不是在一路猜顶，而是在等待上涨动能开始松动。

4. 仓位与风险

仓位不应该从感觉出发，而应该从最大亏损反推。假设第 i 层开空价是 E_i，止损价是 X_i，这一层最多愿意亏 R_i。

单层合约数量： q_i = R_i / (X_i - E_i) 空单浮动盈亏： PnL_i = q_i * (E_i - P) 总最大亏损： MaxLoss = Σ q_i * (X_i - E_i)

这一步是模型的护栏。没有最大亏损预算，阶梯做空很容易退化成逆势加仓，最后被极端拉升打穿。

5. 三种执行方式

同样的价格阶梯，执行方式不同，风险完全不同。

只提醒 价格触及阶梯后只通知，不自动开仓。适合暴涨小币的观察阶段。 风险最低
条件开空 先触及价格层，再等待回落确认或结构转弱，才开一小部分空单。 风险中等
提前挂空 每一层提前挂限价空单，价格涨到就成交。单边拉升时会连续吃单。 风险最高

6. 合约风险

小币合约里，这套模型最怕的不是“方向错一次”，而是方向错了以后还继续加仓。合约市场有保证金和强平机制，极端波动时保证金要求、标记价格、盘口深度和滑点都会让理论亏损变成真实亏损。

强平与保证金

空单遇到继续上涨，保证金会被侵蚀。价格接近强平线时，交易所可能强制平仓。

滑点与盘口

小币深度薄，止损触发价不等于实际成交价。行情越急，成交越可能偏离模型。

资金费率

永续合约持仓有资金费率成本。高热度币种的费用会改变持仓收益结构。

单边挤压

暴涨小币常有连续拉升、插针和逼空。提前挂空容易让所有层同时变成浮亏。
