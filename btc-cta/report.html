方法论

把回测器做成自优化 Harness

源自 OpenAI / Anthropic 的 harness engineering + arXiv 2603.03329 AutoHarness。 核心是 Recursive Self-Improvement 的复利飞轮——但在量化里,它最大的风险是过拟合。

0INITIALIZER 一次性抓数据 → 三段锁死: train / select(OOS) / holdout(只看一次)
↓
1提假设 — LLM/人 只改 EVOLVE-BLOCK 标记区
2回测 — 向量化引擎跑 train + walk-forward
3评估 — 多目标 JSON Sharpe/Calmar/MaxDD(绝不单一 reward)
4选择 — Optuna/Nevergrad + MAP-Elites 保多样性
5固化 — champion 晋升 + 可回滚 → 暴露下一个 gap → 编码回 harness
↺ 复利飞轮 · 然后 HOLDOUT 验收只看一次
① OOS 三段锁死 ② 多目标评估 ③ 沙箱执行 ④ champion 可回滚 ⑤ walk-forward 硬约束

实证教训:即便有两段防护,select 上调出的参数到 holdout 仍过拟合。 holdout 是唯一照妖镜——环境设计 > 聪明的优化。
