# MA 均线策略横向对比 + 交叉测试

> 引擎 `src/backtest_ma_grid.py`(周期×带宽网格)+ `src/backtest_ma_zoo.py`(8 策略交叉测试)。
> 全部仅做多,三段交叉:全样本 2012+ / 2020+ / holdout 2023+。日期 2026-06-07。
> 缘起:reddit 原版是 **MA200±5%**(sell if −5% below MA200, buy if +5% above),MA95±4 是用户变体。

## 一、reddit MA200±5% vs user MA95±4%

| 段 | MA95±4(user) Calmar | MA200±5(reddit) Calmar | 换仓(reddit/user) |
|---|---|---|---|
| 全样本 | **1.60** | 1.24 | 32 / 54 |
| 2020+ | **1.65** | 0.78 | 14 / 22 |
| holdout 2023+ | 0.82 | **0.95** | **8 / 16** |

- MA95±4 在长历史明显更优,且处于网格**平滑高地**(MA95-100/±2-4% 整片 1.5-1.6,非孤立尖峰)。
- 但 holdout 两者打平,MA200±5 凭**更少换仓(8)、更高胜率(75%)**略胜——更钝更省心。

## 二、8 策略交叉测试 · 跨段稳健性(三段 Calmar 最小值,越高越稳)

| 排名 | 策略 | 全样本 | 2020+ | holdout | **最差段** | 全段均值 |
|---|---|---|---|---|---|---|

## 三、结论

1. **用稳健性视角(看最差段,而非最高分),MA95±4 反而是冠军**(最差段 0.82、全段均值 1.36 双第一)。
   它不是某段最强,而是**三段都没掉链子**。这回答了"MA95±4 是否合适":**合适,且是瞎选不出来的稳健解。**
2. **reddit MA200±5 是更省心的次选**:换仓最少、胜率最高、holdout 单段最强(0.95)。钝感 = 操作少 =
   手抖机会少,对真实持有体验更友好(呼应"机械纪律外骨骼"的心性结论)。
3. **我自构思的 6 个更复杂策略,没有一个打败简单的单均线包络带**:
   - 经典金叉死叉(dual_cross)在 BTC 上**最差**(holdout 0.30),多头排列(triple_align)更差(0.25)——
     进场滞后,趋势反转时割在半山腰。
   - keltner/dual_envelope/slope_filter 都不如 user_95_4。
   - **复杂度没换来稳健性。** 这是整个 MA 探索最干净的元结论:**简单稳健 > 复杂精巧。**
4. **网格的过拟合铁证**:全样本最优区(MA95±4)在 holdout 整体塌陷,holdout 高分跑到 MA50±0/MA250±8
   等完全不同位置。**别把历史最优参数当真。**

## 四、一个诚实的不一致(需注意)
`*continuous` 这里最差段 0.75,但 `backtest_v2.py` 里 holdout Calmar 是 **1.13**——因为两处映射定义不同:
- zoo 版:MA 处 0.5 仓 → 频繁微调,换手 973/297 次,成本拖累
- v2 版:MA 处 0 仓、突破才加仓 → 换手低,holdout 1.13
**推荐用 v2 版定义的 continuous。** continuous 对实现细节敏感,这点本身是个警示:连续仓位的换手控制很关键。

## 五、最终选择建议
- **要稳健 + 历史最优**:user MA95±4(或其 v2 continuous 变体)
- **要省心 + 钝感持有**:reddit MA200±5(换仓最少,近年表现反而最好)
- **别碰**:金叉死叉、多头排列等"看起来更专业"的复杂 MA 组合,在 BTC 上更差。
